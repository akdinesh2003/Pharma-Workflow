Workflow Process Details

Complete pharmaceutical workflow management process

Main Workflow Path

1

Initiated

👤 Who: User, Manager, Admin

📝 What: Create new drug batch request

A new pharmaceutical batch request is created with drug name, batch number, and initial comments.

↓
2

Approved

👤 Who: Manager, Admin

✅ What: Review and approve request

Quality assurance team reviews the request. If it meets standards, the request is approved for production.

↓
3

Active

👤 Who: Manager, Admin

🚀 What: Activate for production

The approved batch is activated and production begins. The batch is now in active manufacturing.

↓
4

Inactive

👤 Who: Manager, Admin

✔️ What: Complete and close

Production is completed. The batch is marked as inactive and the workflow is closed.

Alternative Path: Rejection

1

Initiated

Request is created and submitted for review.

↓
❌

Rejected

👤 Who: Manager, Admin

❌ What: Reject request

If the request doesn't meet quality standards or has issues, it is rejected. The workflow ends here.

Modification Option

✏️

Modified

👤 Who: Manager, Admin

✏️ What: Modify request details

⏰ When: At Initiated or Approved stage

Request details (drug name, batch number, comments) can be modified before activation. After modification, the status changes to "Modified" and requires re-approval.

Role Permissions

👤 User

  • ✅ Create new requests
  • ✅ View all requests
  • ✅ View history
  • ❌ Cannot approve/reject
  • ❌ Cannot modify
  • ❌ Cannot activate/complete
  • ❌ Cannot delete

👨‍💼 Manager

  • ✅ All User permissions
  • ✅ Approve requests
  • ✅ Reject requests
  • ✅ Modify requests
  • ✅ Activate requests
  • ✅ Complete requests
  • ❌ Cannot delete

👨‍💻 Admin

  • ✅ All Manager permissions
  • ✅ Delete requests (soft delete)
  • ✅ Full system access

📋 Audit Trail

Every action in the workflow is recorded with:

  • 🕐 Timestamp - When the action occurred
  • 👤 User - Who performed the action
  • 📝 Action - What was done (Created, Approved, Rejected, etc.)
  • 💬 Comments - Additional notes or reasons
  • 🔄 Status Change - Previous status → New status

This ensures complete traceability for regulatory compliance.
